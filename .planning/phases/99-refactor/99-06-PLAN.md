**Current structure of app.rs:**
- Lines 1-111: Type definitions (Focus, SyncMsg, SyncPhase, SyncPhaseKind, ImageLoadMsg, ImageState, ImageLoadState, ArticleReaderState, App struct)
- Lines 113-200: `App::new()` constructor

- Lines ~500-600: Sync-related methods (trigger_sync, poll_sync, start_search, poll_search)
- Lines ~600-700: Navigation and state methods (load_articles, load_sections, open_article, etc.)
- Lines ~700-809: Image loading (spawn_image_load_thread, poll_image_load)

**Decomposition plan:**

1. Create directory `src/app/`

2. Create `src/app/mod.rs` — the App definition and core logic:
   - All type definitions: `Focus`, `SyncMsg`, `SyncPhase`, `SyncPhaseKind`, `ImageLoadMsg`, `ImageState`, `ImageLoadState`, `ArticleReaderState`
   - `App` struct definition (all fields)
   - `App::new()` constructor
   - Sync methods: `trigger_sync()`, `poll_sync()`, `start_search()`, `poll_search()`
   - Navigation methods: `load_articles()`, `load_sections()`, `open_article()`, `filtered_articles()`
   - `mod handlers;` and `mod image_loader;` declarations
   - `pub use` re-exports for all public types
   - The key handling dispatch (`handle_key`) stays here as it delegates to handlers

3. Create `src/app/handlers.rs` — key handling:
   - `handle_calendar_key()`
   - `handle_article_list_key()`
   - `handle_article_reader_key()`
   - `handle_search_key()`
   - `handle_section_picker_key()`
   - These are `pub(crate)` or `pub(super)` methods on `App` — use `impl App` block in this file
   - Imports: `crossterm::event::{KeyCode, KeyEvent}`, `crate::app::*` (or `super::*`)

4. Create `src/app/image_loader.rs` — image loading:
   - `spawn_image_load_thread()` method on `App`
   - `poll_image_load()` method on `App`
   - Image loading logic and thread spawning
   - Imports: `super::*`, `ratatui_image::*`, `std::sync::mpsc`, `std::thread`

5. Delete `src/app.rs`

6. **Handler pattern**: Since key handlers are `impl App` methods, they need access to `self`. In Rust, you can split `impl` blocks across files within the same crate using `impl App { ... }` in each submodule. Each submodule does:
   ```rust
   use super::*;
   
   impl App {
       pub(super) fn handle_calendar_key(&mut self, key: KeyEvent) { ... }
   }
   ```

7. **No circular dependencies.** Direction: `app/*` imports from `db`, `sync` (types). UI imports from `app`. Main imports from all. This is already the existing direction — no change.

8. **Preserve public API:**
   - `crate::app::App` — struct definition in mod.rs, re-exported ✓
   - `crate::app::Focus` — enum in mod.rs ✓
   - `crate::app::SyncMsg` — enum in mod.rs ✓
   - `crate::app::SyncPhase` — struct in mod.rs ✓
   - `crate::app::SyncPhaseKind` — enum in mod.rs ✓
   - `crate::app::ImageLoadMsg` — enum in mod.rs ✓
   - `crate::app::ArticleReaderState` — struct in mod.rs ✓
